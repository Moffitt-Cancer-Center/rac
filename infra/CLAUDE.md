# infra — Tier 2 platform Bicep

**Freshness:** 2026-04-24

## Purpose

Deploys the per-environment Tier 2 platform: networking, identity, Postgres, ACR, Key Vault, Blob, ACA env, Front Door + WAF, App Gateway, Log Analytics, Event Hub, alerts, the Shim ACA app, and two scheduled ACA jobs (graph-sweep, cost-ingest). Tier 3 researcher apps are *not* deployed here — they are provisioned at runtime by the control plane via the Azure SDK.

## Layout

- `main.bicep` composes every module; `main.json` is the compiled ARM output and is checked in.
- `modules/*.bicep` — one module per resource family. Each emits a `.json` sibling.
- `environments/<env>.bicepparam` — env-specific inputs for `dev`, `staging`, `prod`.
- `../scripts/teardown.sh` — `everything off` switch for an environment; nukes RG but preserves soft-deleted KV/secrets for the retention window.

## The two-pass deploy (CRITICAL)

Several resources have circular dependencies that Bicep cannot resolve in a single pass:

1. **Pass 1** — Deploy with `deployRoleAssignments=false` and `deployShimApp=false`. This creates the managed identity, KV, Postgres, ACA env, Shim app registry tables, etc.
2. **Out-of-band step** — Run Postgres bootstrap SQL (`docs/runbooks/bootstrap.md` step 6) which creates the `rac_shim` role (migration 0009). Seed KV secrets and reviewer signing key. Get the managed identity's principal id.
3. **Pass 2** — Deploy with both flags true. `role-assignments.bicep` grants the MI the roles it needs (AcrPull, KV Secrets/Keys User, Storage Blob Data Contributor, etc.); `shim-aca-app.bicep` deploys the shim container with KV references resolved.

Skipping pass 1 → deploying Shim will fail because its MI has no KV access yet.

## Module invariants

- Every resource gets tags from `tags.bicep` (`env`, `cost-center`, `workload=rac`, `managed-by=bicep`). Resources that do not inherit tags must call the helper explicitly.
- KV is created with **purge protection on** and a **90-day soft-delete retention** in non-dev. Any change here must be param-driven (`enablePurgeProtection`, `softDeleteRetentionInDays`) — never hard-coded.
- Network subnets are declared in `network.bicep`; ACA env, Postgres, and private endpoints all reference subnet outputs. Adding a new subnet always goes in this module, never inlined.
- Role IDs (built-in Azure roles) come from `role-ids.bicep` — never hard-code a role GUID elsewhere.

## Scheduled jobs

- `graph-sweep-job.bicep` — runs the `graph_sweep_cli` once a day; queries MS Graph for deactivated PIs and flags their apps for ownership-transfer.
- `cost-ingest-job.bicep` — runs `cost_ingest_cli` to pull Azure Cost Management snapshots into `cost_snapshot`.

Both jobs are ACA jobs (not Functions) and use the same managed identity as the control plane.

## Bicep vs ARM

Only touch `.bicep`. The `.json` files are regenerated by `az bicep build` and committed. The GHA workflow (`.github/workflows/infra-deploy.yml`) deploys from the JSON for reproducibility.

## Test & verification

Unit-test Bicep with `az deployment sub validate` and `what-if` before merge. The infra-deploy workflow runs `what-if` on PR and deploys on merge to `main` for each environment with manual approval gates.
